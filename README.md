# Black Heritage Friesian Stud

## Учебный веб-проект для производственной практики

Проект выполнен в рамках профессионального модуля **ПМ.09 «Проектирование, разработка и оптимизация веб-приложений»**.

**Black Heritage Friesian Stud** — это адаптивный сайт для элитного племенного завода фризских лошадей. Проект разработан как демонстрационное веб-приложение с клиентской частью, серверной обработкой формы, подключением к базе данных и простой административной страницей для просмотра заявок.

---

## Общая задумка проекта

Основная идея проекта — создать не обычный информационный сайт, а визуально выразительный и премиальный веб-ресурс для конного хозяйства, которое занимается разведением, продажей и презентацией фризских лошадей.

Фризская лошадь ассоциируется с **силой, грацией, породностью, статусом и эффектной внешностью**. Поэтому сайт был задуман в соответствующей стилистике: с тёмной атмосферной палитрой, золотыми акцентами, крупными изображениями, спокойной композицией и ощущением закрытого элитного клуба.

Проект должен показать, как с помощью базовых веб-технологий можно создать сайт, который выполняет сразу несколько задач:

* презентует бренд;
* рассказывает о деятельности хозяйства;
* демонстрирует лошадей;
* собирает заявки;
* сохраняет данные в базу;
* позволяет администратору просматривать обращения пользователей.

---

Концепция сайта строится вокруг образа **премиального племенного завода**.

Визуально проект должен вызывать ассоциации с:

* частными просмотрами;
* породным разведением;
* индивидуальным подбором лошадей;

При разработке сайта акцент был сделан на:

* **удобную структуру разделов**;
* **крупные фотографии лошадей**;

## Целевая аудитория

Сайт ориентирован на пользователей, которые интересуются фризскими лошадьми и могут быть потенциальными клиентами племенного хозяйства.

К целевой аудитории относятся:

* частные покупатели, заинтересованные в приобретении фризской лошади;
* владельцы конных клубов;
* заводчики и специалисты по разведению;
* люди, интересующиеся элитными породами лошадей;
* клиенты, которым нужен индивидуальный подбор лошади;
* посетители, желающие записаться на частный просмотр.

Именно поэтому сайт выполнен не как простой каталог, а как презентационный ресурс, который должен формировать доверие и подчёркивать статус хозяйства.

## Пользовательский сценарий

Пользователь заходит на сайт и сразу попадает на главный экран, где видит название хозяйства и общее позиционирование проекта. Далее он может перейти к разделу с описанием завода, изучить направление работы, посмотреть доступных лошадей и ознакомиться с производителями.

Особое внимание уделено разделу **«На продажу»**. В нём пользователь может выбрать интересующую лошадь и посмотреть подробную информацию:

* фотографии;
* характеристики;
* описание;
* назначение;
* возможность оставить заявку.

После этого пользователь может оставить заявку на частный просмотр, указав свои контактные данные и цель обращения.

После отправки формы данные сохраняются в базу данных. Администратор может открыть страницу **`admin.php`** и увидеть список поступивших заявок.

Таким образом, сайт имитирует полный цикл работы небольшого веб-приложения:

1. пользователь просматривает информацию;
2. выбирает интересующую лошадь;
3. отправляет заявку;
4. сервер обрабатывает данные;
5. информация сохраняется в базе;
6. администратор просматривает заявки.

**Задача главного экрана** — заинтересовать пользователя и побудить его продолжить просмотр сайта.

Раздел **«О нас»** рассказывает о концепции племенного завода и создаёт первое доверие к проекту.

## Раздел «На продажу»

Раздел **«На продажу»** является одним из ключевых элементов сайта. Он выполнен как интерактивный каталог лошадей.

Пользователь может выбрать лошадь из списка, после чего на странице меняются:

* фотографии выбранной лошади;
* характеристики;
* описание;
* информация о назначении;

Такой подход делает раздел более живым и удобным, чем обычный статичный список карточек. Пользователь может изучить каждую лошадь отдельно и сразу перейти к отправке заявки.

Для такого сайта этот раздел особенно важен, потому что в сфере разведения большое значение имеют происхождение, экстерьер, достижения и репутация производителей.

На сайте реализована форма заявки на частный просмотр и подбор лошади.

Форма собирает следующие данные:

* имя;
* телефон;
* email;
* интересующая лошадь;
* цель подбора;
* формат просмотра;
* удобная дата;
* комментарий;
* согласие на обработку персональных данных.

Форма является важной частью проекта, так как делает сайт не просто статичной страницей, а полноценным веб-приложением с серверной обработкой данных.

После отправки данные проходят проверку и сохраняются в базу данных **MySQL**.

## Размещение на хостинге

Проект размещён на бесплатном PHP-хостинге **InfinityFree**.

Для размещения были выполнены следующие действия:

1. создан поддомен **`blackheritage.xo.je`**;
2. файлы проекта загружены в папку **`htdocs`**;
3. создана база данных **MySQL**;
4. через **phpMyAdmin** выполнен SQL-скрипт создания таблицы **`viewing_requests`**;
5. в файле **`includes/db.php`** указаны данные подключения к базе на хостинге;
6. проверена отправка формы;
7. проверен вывод заявки на странице **`admin.php`**.

## Безопасность

В проекте реализованы базовые меры безопасности:

* данные формы очищаются перед обработкой;
* email проверяется на корректность;
* значения списков проверяются на допустимые варианты;
* дата не может быть прошедшей;
* запись в базу выполняется через подготовленные SQL-запросы;
* данные в админке выводятся через **`htmlspecialchars`**;
* реальные данные подключения к базе не публикуются в GitHub.
